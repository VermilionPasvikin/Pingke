# 评课系统版本历史

## 版本号规范

### 版本号格式

评课系统使用语义化版本号（Semantic Versioning）进行版本管理，版本号格式如下：

```
X.Y.Z[-label]
```

其中：

- `X` 为主版本号：当进行不兼容的API更改时递增
- `Y` 为次版本号：当以向后兼容的方式添加功能时递增
- `Z` 为修订号：当进行向后兼容的错误修复时递增
- `[-label]` 为预发布版本标签（可选）

### 正式版与测试版区分

#### 正式版

正式版版本号格式为 `X.Y.Z`，例如：

- `1.0.0`
- `2.1.3`

#### 测试版

测试版版本号包含预发布标签，格式为 `X.Y.Z[-label]`，例如：

- `1.0.0-beta.1`
- `2.0.0-rc.2`

预发布标签类型：

- `-alpha`：内部测试版本
- `-beta`：公开测试版本
- `-rc`：候选发布版本

## 版本更新记录规范

每次版本更新需按照以下格式记录：

### 版本条目格式

```markdown
### X.Y.Z[-label]（发布日期：YYYY-MM-DD）

#### 功能新增
- [功能点描述]

#### 问题修复
- [修复内容描述]

#### 功能优化
- [优化内容描述]

#### 重要变更
- [重要变更描述]

#### 兼容性说明
- 支持微信小程序环境
- [兼容性相关说明]
```

### 记录要点

1. **版本号**：使用规范的语义化版本号，并明确区分正式版与测试版
2. **发布日期**：使用ISO标准日期格式 YYYY-MM-DD
3. **分类记录**：按照功能新增、问题修复、功能优化、重要变更、兼容性说明进行分类
4. **描述清晰**：简要说明变更内容，突出用户感知的变化
5. **版本顺序**：新版本在上，旧版本在下，保持倒序排列
